Developing rieszboost

├── python/
│   ├── rieszboost/              # the library
│   │   ├── __init__.py            # public API surface, lazy imports
│   │   ├── estimator.py           # RieszBooster (sklearn BaseEstimator)

│   │   ├── losses.py              # LossSpec protocol + four losses
│   │   ├── diagnostics.py         # diagnose() + Diagnostics dataclass

Run tests

Rscript -e '
  Sys.setenv(RETICULATE_PYTHON = file.path(getwd(), ".venv/bin/python"))
  pkgload::load_all("r/rieszboost")
  testthat::test_dir("r/rieszboost/tests/testthat")
'

xgboost requires libomp on macOS: brew install libomp once.

Tracer

The user writes m(z, alpha) with +, -, scalar *, and calls to alpha(...). The library passes a Tracer in as alpha; each call returns a single-term LinearForm, and Python’s operator overloading composes them. The result has .as_pairs() returning [(coef, point), ...], which build_augmented consumes to construct the augmented dataset. Operations outside the linear-form algebra (squaring, integrals, constants) raise at trace time. See python/rieszboost/tracer.py.

Adding an estimand factory

A factory returns an Estimand. Pattern from estimand.py:

def MyEstimand(param, treatment="a", covariates=("x",)) -> Estimand:
    cov = tuple(covariates)
    def m(z, alpha):
        x_kwargs = {k: z[k] for k in cov}
        return alpha(**{treatment: param, **x_kwargs})
    return Estimand(feature_keys=(treatment, *cov), m=m, name=f"MyEstimand({param})")

Then:

  1. Export from python/rieszboost/__init__.py (add to imports and __all__).
  2. Add an R wrapper in r/rieszboost/R/rieszboost.R and @export it.
  3. Update docs/estimands.qmd with a row in the built-ins table and a runnable example in both languages.
  4. Add a test under python/tests/.
  5. Add a worked example under examples/ (per the per-estimand example rule in CLAUDE.md).

The pre-commit hook blocks step (1)/(2) without step (3).

Adding a LossSpec

Implement the LossSpec protocol from losses.py. SquaredLoss is the shortest template; KLLoss shows how to handle a non-identity link.

  1. Add the class to python/rieszboost/losses.py.
  2. Export from __init__.py.
  3. Add an R wrapper.
  4. Add a runnable example to docs/backends-and-losses.qmd.
  5. Test in python/tests/test_bregman.py.

Adding a backend

Implement the Backend protocol from python/rieszboost/backends/base.py. One method, fit_augmented(...), returning a FitResult whose predictor implements predict_eta and predict_alpha. XGBoostBackend and SklearnBackend are reference implementations.

The backend sees an AugmentedDataset and a LossSpec. It does not need to know about the estimand or the tracer.

Living-doc rule

Public-API changes update both README.md and the relevant docs/*.qmd page in the same commit. On bilingual pages, update both the Python and R tabs.

The pre-commit hook at .githooks/pre-commit enforces this. Activate per clone:

git config core.hooksPath .githooks

If any of these files are staged:

  • python/rieszboost/{__init__,estimator,estimand,augmentation,losses,diagnostics,tracer}.py
  • python/rieszboost/backends/*.py
  • r/rieszboost/R/*.R

then at least one of these must also be staged:

  • docs/*.qmd
  • docs/_quarto.yml
  • README.md

The hook also greps the staged diff for two doc anti-patterns and blocks if it finds them: design-decision metacommentary (sentences describing the API’s negative space, e.g. what’s omitted, deferred, or rejected) and AI-flavored hedges (editorial framing, sweeping qualifiers, ornamental noun phrases). The hook’s error message lists concrete bad/good rewrite pairs. Match the ngboost user guide tone: direct, instructive, short.

For internal refactors that don’t touch any signature, test additions, or comment fixes: git commit --no-verify.

Local docs render

quarto preview docs/        # live-reload at http://localhost:port
quarto render docs/         # one-shot build to docs/_site/
open docs/_site/index.html

Renders take ~30s with the committed _freeze/ cache, ~3 minutes cold. Editing a single .qmd only re-renders that page.

CI

The workflow at .github/workflows/docs.yml runs on push to main when any of docs/, python/, r/, examples/, or the workflow itself changes. It checks out the repo (restoring _freeze/), installs Python 3.11 + rieszboost + matplotlib, installs R + knitr + rmarkdown + reticulate, sets up Quarto, renders, and publishes _site/ to the gh-pages branch.

The site lives at https://alejandroschuler.github.io/rieszboost/.
