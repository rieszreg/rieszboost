scikit-learn integration

RieszBooster is an sklearn.base.BaseEstimator. Anything in sklearn.model_selection works on it without adapter code: cross-fitting, hyperparameter search, pipelines, model cloning. There is intentionally no bespoke crossfit() function — the design rule is to lean on scikit-learn’s well-tested machinery rather than reinvent it.

Cross-fitting for downstream inference

Plug-in \(\hat\alpha\) in a one-step / TMLE / DML estimator must be out-of-fold to avoid bias from over-fitting. cross_val_predict does this for any sklearn estimator.

  • Python
  • R
import numpy as np, pandas as pd
from sklearn.model_selection import cross_val_predict
from rieszboost import RieszBooster, ATE

rng = np.random.default_rng(0)
n = 2000
x = rng.uniform(0, 1, n)
pi = 1 / (1 + np.exp(-(-0.02 * x - x**2 + 4 * np.log(x + 0.3) + 1.5)))
a = rng.binomial(1, pi)
df = pd.DataFrame({"a": a.astype(float), "x": x})

booster = RieszBooster(
    estimand=ATE(),
    n_estimators=2000,
    early_stopping_rounds=20,
    validation_fraction=0.2,
    learning_rate=0.05,
    max_depth=4,
)

alpha_oof = cross_val_predict(booster, df, cv=5)
alpha_oof.shape, alpha_oof[:5]
((2000,), array([-3.6196816, -1.6876634, -1.0861515, -1.0516312,  1.2505229],
      dtype=float32))
set.seed(0)
n  <- 2000
x  <- runif(n)
pi <- 1 / (1 + exp(-(-0.02 * x - x^2 + 4 * log(x + 0.3) + 1.5)))
a  <- rbinom(n, 1, pi)
df <- data.frame(a = as.numeric(a), x = x)

# The cleanest cross-fit from R is to call the sklearn function via
# reticulate, passing the underlying Python estimator object. The R6 wrapper
# exposes it as `$py`.
sk_ms <- reticulate::import("sklearn.model_selection", convert = FALSE)
booster <- RieszBooster$new(
  estimand = ATE(),
  n_estimators = 2000L,
  early_stopping_rounds = 20L,
  validation_fraction = 0.2,
  learning_rate = 0.05,
  max_depth = 4L
)
alpha_oof <- as.numeric(reticulate::py_to_r(
  sk_ms$cross_val_predict(booster$py, reticulate::r_to_py(df), cv = 5L)
))
length(alpha_oof); head(alpha_oof)
[1] 2000
[1]  1.301584  6.205725  2.930262 -3.024208  1.465699 -1.293439

Each fold runs its own internal early-stopping split — there is nothing to thread between folds.

Hyperparameter search with GridSearchCV

Score is negative held-out Riesz loss (sklearn convention: higher is better), so GridSearchCV selects the configuration with the smallest loss without any sign-flipping on your end.

  • Python
  • R
from sklearn.model_selection import GridSearchCV

grid = GridSearchCV(
    RieszBooster(
        estimand=ATE(),
        n_estimators=300,
        early_stopping_rounds=20,
        validation_fraction=0.2,
    ),
    param_grid={"learning_rate": [0.03, 0.05, 0.1], "max_depth": [3, 4, 5]},
    cv=3,
    n_jobs=1,
).fit(df)

print("best params:", grid.best_params_)
best params: {'learning_rate': 0.05, 'max_depth': 3}
print(f"best score (neg Riesz loss): {grid.best_score_:.4f}")
best score (neg Riesz loss): 5.8153
best = grid.best_estimator_  # already a refitted RieszBooster
alpha_hat = best.predict(df)
alpha_hat[:5]

What composability buys you

Because RieszBooster is a regular BaseEstimator:

  • clone(booster) returns a fresh unfitted copy with the same hyperparameters — useful when you want to fit the same configuration on multiple data subsets.
  • Pipeline can wrap a feature-engineering step in front of the booster.
  • HalvingGridSearchCV, RandomizedSearchCV, BayesSearchCV (skopt) all work without modification.
  • get_params() / set_params() introspect/mutate the configuration; this is what the search classes use under the hood.

The general principle: anywhere scikit-learn’s documentation says “any sklearn estimator,” RieszBooster slots in.
